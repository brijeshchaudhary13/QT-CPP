# 3. Generated Source Files

## main.cpp

```cpp
#include <QApplication>
#include "mainwindow.h"

int main(int argc, char *argv[])
{
    QApplication app(argc, argv);

    MainWindow window;
    window.show();

    return app.exec();
}
```

---

# Understanding Every Line

## Include QApplication

```cpp
#include <QApplication>
```

Provides the GUI application object.

Responsibilities include:

* Event loop
* Window management
* Clipboard
* Style
* Fonts
* Mouse and keyboard events

---

## Include MainWindow

```cpp
#include "mainwindow.h"
```

Includes the declaration of our main window.

---

## main()

```cpp
int main(int argc, char *argv[])
```

Every C++ application starts here.

Qt is no different.

---

## Create QApplication

```cpp
QApplication app(argc, argv);
```

This object initializes the Qt GUI framework.

Internally it:

* Initializes platform plugins.
* Creates the event dispatcher.
* Loads fonts.
* Initializes styles.
* Parses command-line arguments.
* Prepares the event loop.

There should normally be **exactly one** `QApplication` object in a Widgets application.

---

## Create Main Window

```cpp
MainWindow window;
```

Constructs the application's main window.

At this point, the window exists in memory but is **not yet visible**.

---

## Show the Window

```cpp
window.show();
```

Requests that the operating system display the window.

---

## Start the Event Loop

```cpp
return app.exec();
```

This is one of the most important lines in Qt.

`exec()` starts the event loop.

Without it:

* Buttons won't respond.
* Mouse events won't work.
* Keyboard input won't work.
* Timers won't fire.
* Paint events won't occur.

We'll study the event loop deeply in **Chapter 28**.

# 4. mainwindow.h

Typical generated code:

```cpp
#ifndef MAINWINDOW_H
#define MAINWINDOW_H

#include <QMainWindow>

QT_BEGIN_NAMESPACE
namespace Ui {
class MainWindow;
}
QT_END_NAMESPACE

class MainWindow : public QMainWindow
{
    Q_OBJECT

public:
    explicit MainWindow(QWidget *parent = nullptr);
    ~MainWindow();

private:
    Ui::MainWindow *ui;
};

#endif
```

---

## Important Points

### Inheritance

```cpp
class MainWindow : public QMainWindow
```

Our class inherits all the functionality of `QMainWindow`.

---

### Q_OBJECT

```cpp
Q_OBJECT
```

Enables:

* Signals
* Slots
* Properties
* Meta-object system

Without it, many Qt features won't work.

We'll explore it in **Chapters 11 and 12**.

---

### UI Pointer

```cpp
Ui::MainWindow *ui;
```

Points to the object generated from `mainwindow.ui`.

---

# 5. mainwindow.cpp

Typical generated code:

```cpp
#include "mainwindow.h"
#include "ui_mainwindow.h"

MainWindow::MainWindow(QWidget *parent)
    : QMainWindow(parent),
      ui(new Ui::MainWindow)
{
    ui->setupUi(this);
}

MainWindow::~MainWindow()
{
    delete ui;
}
```

---

## setupUi()

```cpp
ui->setupUi(this);
```

This function is generated by `uic`.

It creates:

* Buttons
* Labels
* Layouts
* Menus
* Toolbars
* Status bar
* Central widget

based on `mainwindow.ui`.

# 6. Qt Console Application

A console application doesn't require a graphical user interface.

Typical uses:

* Command-line tools
* File converters
* Data processing
* Batch jobs
* Servers

---

## Generated main.cpp

```cpp
#include <QCoreApplication>
#include <QDebug>

int main(int argc, char *argv[])
{
    QCoreApplication app(argc, argv);

    qDebug() << "Hello Qt!";

    return app.exec();
}
```

---

## Why QCoreApplication?

Unlike `QApplication`, `QCoreApplication` provides:

* Event loop
* Timers
* Signals/Slots
* File handling
* Networking

but **no GUI support**.

---

# Widgets vs Console

| Feature    | QApplication | QCoreApplication |
| ---------- | ------------ | ---------------- |
| GUI        | ✔            | ✘                |
| Event Loop | ✔            | ✔                |
| Widgets    | ✔            | ✘                |
| Network    | ✔            | ✔                |
| Timers     | ✔            | ✔                |

# 7. Qt Quick (QML) Application

Qt Quick is used to create modern, animated user interfaces.

Common uses:

* Automotive HMIs
* Embedded devices
* Touchscreen applications
* Mobile apps

---

## Generated main.cpp

```cpp
#include <QGuiApplication>
#include <QQmlApplicationEngine>

int main(int argc, char *argv[])
{
    QGuiApplication app(argc, argv);

    QQmlApplicationEngine engine;
    engine.load(QUrl(QStringLiteral("qrc:/Main.qml")));

    if (engine.rootObjects().isEmpty())
        return -1;

    return app.exec();
}
```

---

# Why QGuiApplication?

`QGuiApplication` provides:

* Event loop
* Fonts
* Clipboard
* Screen information
* Input events

but **does not include the Widgets module**.

# 8. CMake Project

Modern Qt applications use CMake.

Example:

```cmake
cmake_minimum_required(VERSION 3.21)

project(MyApp LANGUAGES CXX)

find_package(Qt6 REQUIRED COMPONENTS Widgets)

qt_add_executable(MyApp
    main.cpp
    mainwindow.cpp
    mainwindow.h
    mainwindow.ui
)

target_link_libraries(MyApp PRIVATE
    Qt6::Widgets
)
```

# 13. Best Practices

* Create only one `QApplication` object.
* Always call `app.exec()`.
* Prefer CMake for new projects.
* Keep `main()` simple.
* Place business logic in separate classes.
* Do not perform heavy initialization in the UI thread before showing the main window.

---

# 14. Common Mistakes

* Forgetting `app.exec()`.
* Creating multiple `QApplication` instances.
* Putting all application logic inside `main()`.
* Editing generated `ui_*.h` files.
* Blocking the event loop with long-running tasks.

# 17. Revision Notes

* `QApplication` is required for Widgets applications.
* `QCoreApplication` is used for non-GUI applications.
* `QGuiApplication` is used for GUI applications without Widgets.
* `app.exec()` starts the event loop.
* `setupUi()` builds the widget hierarchy from the `.ui` file.
* CMake is the preferred build system for Qt 6.
* Keep `main()` minimal and delegate work to dedicated classes.
